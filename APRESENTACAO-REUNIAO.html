Parte 2 — A EstruturaO "Segundo Cérebro" — Como Organizaram

Este é o coração de tudo. Um repositório GitHub que serve como memória permanente dos agentes. Sem ele, o agente esquece tudo ao fechar a sessão. Com ele, aprende e evolui.

No final do dia, o grande segredo de tudo isso é você ter esse segundo cérebro. Porque quando você transforma contexto, habilidades numa coisa para agentes, você roda isso em qualquer ferramenta. — Bruno Okamoto

Por que isso é genial

O agente aprende o padrão UMA vez e navega qualquer nível da empresa. É como um organograma que a IA consegue ler e entender sozinha.

Conceitos Fundamentais

Skill = Receita automatizada

Uma tarefa padronizada que o agente executa sempre do mesmo jeito. Exemplos: relatorio-vendas, follow-up-leads, analise-criativos, criar-skill (o agente cria novas skills!).

Cron = Skill que roda sozinha

Agendamento automático. Exemplos: relatório de vendas todo dia 8h, análise de ads toda noite, sync com GitHub meia-noite, heartbeat às 6h.

Memória em 2 níveis

Local (volátil): notas do dia, rascunhos. Morre se reconfigurar o agente.

Repositório GitHub (permanente): fonte de verdade. Versionado, auditável, editável pela equipe.

"Se importa, escreve no repositório. O que está só na memória local, morre com o agente."

Permissionamento por camadas

Cada agente só acessa sua área. Cada pessoa só interage com agentes permitidos. Ações externas (email, post) sempre pedem confirmação humana.

Parte 6 — Como FuncionaO Dia a Dia na Prática

Esta é a parte mais importante pra entender: como a equipe realmente interage com os agentes no dia a dia?

O que é o "Segundo Cérebro" (explicação simples)

O problema

Quando você conversa com o ChatGPT, ele esquece tudo quando fecha a janela. Todo dia é como treinar um estagiário do zero.

A solução: o Segundo Cérebro

É uma pasta com arquivos de texto organizados que contém tudo sobre a empresa: quem são as pessoas, processos, decisões, lições, métricas.

O agente lê esses arquivos antes de fazer qualquer coisa — como se um funcionário lesse o manual inteiro da empresa em 2 segundos toda manhã.

E toda vez que descobre algo novo, ele escreve de volta nos arquivos. O conhecimento acumula como juros compostos.

Não é um software. Não é uma plataforma. São arquivos de texto organizados de um jeito que a IA navega sozinha.

Por que GitHub? Pode ser outra coisa?

OpçãoFunciona?Por quê
GitHub (privado)Melhor opçãoVersionamento automático (histórico de tudo), o OpenClaw já sabe ler/escrever, múltiplos agentes acessam sem conflito, interface web pra equipe editar

Ponto importante: O GitHub é privado — ninguém de fora vê. E a equipe NÃO precisa abrir o GitHub nunca. Ele é o "backend" (a memória). A interface de uso é o Discord/Telegram/WhatsApp que vocês já usam.

Como fica mais inteligente com o tempo?

1. Memória acumulativa

Cada conversa, cada relatório, cada decisão é salva no cérebro. Amanhã o agente sabe o que aconteceu hoje. Mês que vem ele sabe o padrão do mês passado.

2. Lições aprendidas

Quando você corrige o agente ("não, faz assim"), ele registra como lição em lessons.md. Na próxima vez, faz do jeito certo.

3. Skills que evoluem

Quando o agente detecta que você pede a mesma coisa várias vezes, ele sugere: "Percebi que você pediu esse relatório 3 vezes esta semana. Quer que eu crie uma skill pra isso? Assim roda com uma frase ou vira automático."

4. Decisões registradas

Toda decisão estratégica fica em decisions.md. Quando surgir uma decisão parecida no futuro, o agente consulta: "Em março vocês decidiram X por causa de Y. Querem manter ou mudar?"

5. Contexto de cada pessoa

O agente pessoal aprende como cada sócio gosta de receber informação. Pedro prefere bullet points? Michel gosta de detalhes? Luiz quer só o número? Cada um recebe do jeito que funciona melhor pra ele.

Como os sócios "alimentam" o cérebro?

Resposta curta: Não precisam. O agente alimenta sozinho a partir das conversas e tarefas do dia a dia.

O mais importante

O cérebro se alimenta pela interação natural com os agentes. Não é mais uma ferramenta que a equipe precisa atualizar manualmente (como Notion/Trello). O agente faz isso sozinho como parte do trabalho dele.
