# SPIN Selling — Scripts Canônicos Expandidos

**Tipo:** metodologia (scripts práticos)
**Pré-requisito:** ler `spin-selling-principios.md` e `spin-selling-execucao.md` antes.

---

## Como usar este documento

Complementa `spin-selling-execucao.md` com **scripts mais elaborados, em PT-BR autêntico, adaptados ao contexto Pinguim** (mentoria, infoproduto, criação de conteúdo, marketing digital).

Cada script vem com:
- **Tipo SPIN** (S, P, I, N)
- **Quando usar**
- **Adaptação Pinguim**
- **Variações**

---

## S — Perguntas de Situação (use POUCAS)

**Regra de ouro:** se a resposta está no LinkedIn, no site da empresa, ou numa rápida pesquisa do Google, **NÃO PERGUNTE**. Cada pergunta de Situação que você faz e poderia ter pesquisado **queima credibilidade**.

### Script S.1 — Validação de baseline
```
Pelo que vi no seu Instagram, você posta há uns 3 anos sobre [nicho]. Tô certo, ou peguei errado?
```

**Quando usar:** abertura de call. Demonstra que pesquisou, valida fato com mínimo esforço.

**Adaptação Pinguim:**
- "Vi que você fez o Lo-fi Desafio em janeiro. Esse foi seu primeiro contato com a gente?"
- "Você está há quanto tempo no nicho de [X]?"

### Script S.2 — Contexto operacional
```
Hoje, como funciona seu funil de vendas? Você usa anúncios pagos, faz lançamento, vende perpétuo, ou é mistura?
```

**Quando usar:** quando você precisa entender estrutura atual antes de propor solução.

**Princípio:** **uma pergunta de Situação por chamada** geralmente é o suficiente. Mais que isso vira interrogatório.

### Script S.3 — Métricas atuais (quando relevantes)
```
Sem precisar entrar em detalhes confidenciais, você está em qual faixa de faturamento mensal hoje? Pra eu calibrar o nível de conversa.
```

**Quando usar:** quando o tipo de oferta depende muito do tamanho do negócio do cliente.

**Adaptação Pinguim:** "Você fatura em qual faixa: até R$ 10K/mês, R$ 10-50K, R$ 50K+?"

## P — Perguntas de Problema

**Função:** fazer cliente identificar e verbalizar dificuldades. Cada problema descoberto vira "necessidade implícita".

### Script P.1 — Problema declarado
```
Você poderia ter procurado solução há 6 meses, há 1 ano. Por que está procurando agora?
```

**Quando usar:** abertura. Identifica o gatilho que trouxe o cliente.

**Princípio:** "agora" geralmente revela a dor real. Cliente que veio "agora" tem urgência maior que cliente que "está pesquisando".

### Script P.2 — Frustração específica
```
O que tem te incomodado mais no [área específica do negócio]?
```

**Variações por nicho Pinguim:**
- Pra criador: "o que tá te incomodando mais na sua presença digital hoje?"
- Pra dono de e-commerce: "qual gargalo do seu funil tá te tirando o sono?"
- Pra mentor/expert: "qual parte do seu negócio você gostaria de automatizar?"

### Script P.3 — Comparativo com pares
```
Quando você olha pra outros [criadores / experts / empresas] do seu nicho, o que tá funcionando pra eles que ainda não tá pra você?
```

**Por que funciona:** força cliente a confrontar comparação. Frequentemente revela inveja produtiva (ex: concorrente cresceu mais).

### Script P.4 — Tentativas anteriores
```
O que você já tentou pra resolver isso? Funcionou parcialmente, não funcionou, parou no meio?
```

**Princípio:** descobre cicatrizes. Cliente queimado por curso anterior tem objeção específica que você precisa endereçar.

### Script P.5 — Onde "vaza dinheiro"
```
Onde você sente que está deixando dinheiro na mesa hoje?
```

**Quando usar:** com clientes mais comerciais (donos de negócio).

**Variações:**
- "Que parte do funil te frustra mais quando você olha as métricas?"
- "Se você pudesse mover UMA coisa do seu negócio pra ela funcionar perfeito, o que seria?"

## I — Perguntas de Implicação (AS MAIS IMPORTANTES)

**Função crítica:** transformar problema reconhecido em **dor sentida**. Vendedores top fazem 4x mais perguntas de Implicação que vendedores médios.

### Categoria — IMPLICAÇÃO POR TEMPO

### Script I.1 — Custo de tempo
```
Quantas horas por semana você gasta com [tarefa que o produto resolve]?
```

### Script I.2 — Projeção do custo
```
Se você somar isso ao longo dos próximos 6 meses, quanto tempo é? E ao longo do ano?
```

### Script I.3 — Custo de oportunidade
```
Esse tempo, se fosse seu pra fazer outra coisa, em que você investiria? Mais marketing? Estratégia? Família?
```

**Princípio:** transforma "tempo gasto" em "tempo perdido". Tempo perdido dói mais.

---

### Categoria — IMPLICAÇÃO POR DINHEIRO

### Script I.4 — Custo direto da inação
```
Se essa situação continuar mais 6 meses, o que isso te custa em dinheiro concreto? Em receita perdida, em cliente que não fechou?
```

### Script I.5 — Comparação com concorrente
```
Enquanto isso tá assim no seu negócio, o que os concorrentes estão fazendo? Eles estão crescendo, te ultrapassando?
```

**Adaptação Pinguim:** "Quanto faturamento você acha que perdeu nos últimos 12 meses por não ter [estrutura X]?"

### Script I.6 — Efeito composto
```
Se a tendência se mantiver, em 12 meses como tá sua situação? Em 24?
```

**Princípio:** força cliente a projetar. Projeção de futuro ruim acelera decisão.

---

### Categoria — IMPLICAÇÃO POR EMOCIONAL/PESSOAL

### Script I.7 — Saúde mental
```
Como esse problema afeta seu humor? Você leva pra casa? Atrapalha o sono?
```

**Quando usar:** quando cliente é dono de negócio sob estresse.

### Script I.8 — Família/relacionamento
```
Esse problema chega na sua família? Você tem que dar conta de tempo de trabalho que vira tempo roubado dos seus?
```

**Princípio:** dor emocional/pessoal frequentemente é mais forte que dor financeira. Vendedor maduro chega lá quando faz sentido.

### Script I.9 — Auto-estima profissional
```
Quando você compara onde você está hoje com onde imaginou estar nessa fase, como é? Tá no caminho? Atrasado?
```

**Quando usar:** com clientes de meia-idade refletindo sobre carreira.

---

### Categoria — IMPLICAÇÃO POR REPUTAÇÃO

### Script I.10 — Imagem profissional
```
Como essa situação afeta como o mercado te enxerga? Concorrentes, clientes, parceiros?
```

### Script I.11 — Time/equipe
```
Sua equipe percebe esse problema? Como eles reagem?
```

**Adaptação Pinguim:** "Seus alunos sentem essa lacuna no seu produto?"

## N — Perguntas de Necessidade-de-Solução (need-payoff)

**Função:** levar o cliente a verbalizar **o valor da solução com as próprias palavras**. Quando cliente fala o benefício, ele se convence sozinho.

### Script N.1 — Valor da solução
```
Se você resolvesse esse problema dos [X] que você descreveu, o que mudaria pra você?
```

### Script N.2 — Quantificação do benefício
```
Se a gente conseguisse mover [métrica] em [Y%], quanto isso significaria pra você por mês? Por ano?
```

### Script N.3 — Imaginação de cenário positivo
```
Imagina que daqui a 6 meses você não tem mais esse problema. Como tá sua vida nesse cenário? O que você fez com [tempo/dinheiro/energia liberados]?
```

**Princípio:** força cliente a imaginar futuro melhor. Imaginação positiva gera desejo.

### Script N.4 — Valor de cada componente
```
Se eu te ajudasse a [resolver X específico], isso seria útil? Como?
```

**Quando usar:** validar valor de cada parte do produto.

**Adaptação Pinguim:** "Se você tivesse acesso a uma comunidade de outros [perfil parecido], seria útil pra ti? Por quê?"

### Script N.5 — Urgência interna
```
Quando você gostaria de estar nesse lugar [resultado X]? Em quanto tempo?
```

**Princípio:** cria deadline interno. Cliente verbaliza urgência sem pressão do vendedor.

### Script N.6 — Inversão suave
```
Vamos imaginar o contrário: se a gente NÃO resolvesse isso, o que aconteceria? Você ficaria como?
```

**Quando usar:** depois de Implicação. Reforça custo da inação.

## Sequências canônicas (combinação SPIN)

### Sequência 1 — Descoberta clássica (call de 30min)

```
S.2 — "Como funciona seu funil de vendas hoje?"
P.1 — "Por que você está procurando solução agora?"
P.2 — "O que está te incomodando mais no [área]?"
I.1 — "Quanto tempo isso te consome por semana?"
I.4 — "Se continuar 6 meses, quanto te custa em dinheiro?"
I.7 — "Como isso afeta sua família, seu humor?"
N.1 — "Se a gente resolvesse, o que mudaria pra você?"
N.2 — "Em números, o que isso valeria por ano?"
N.5 — "Quando você gostaria de estar nesse cenário?"
```

### Sequência 2 — Recuperação de lead frio (call de 15 min)

```
P.1 — "Faz tempo que a gente conversou. O que mudou desde lá?"
I.6 — "Olhando agora, em 6 meses como você projeta sua situação?"
N.3 — "Se a gente conseguisse resolver agora, como tá sua vida em 6 meses?"
[se há sinal positivo] N.5 — "Quando você gostaria de começar?"
```

### Sequência 3 — Quebra de objeção de preço

```
[cliente: "tá caro"]

I.4 — "Antes de falar de preço — quanto esse problema tá te custando hoje em dinheiro?"
N.2 — "Se a gente resolvesse, quanto valeria por mês pra ti?"
[se cliente verbalizou número alto]
N.6 — "Então a gente tá falando de R$ X de upside e R$ Y de investimento. Como você compara?"
```

## Princípios universais por trás dos scripts SPIN

### 1. Pergunte 4x mais Implicação que Situação
Vendedor top: 4 Implicações pra cada 1 Situação. Vendedor ruim: o oposto.

### 2. Implicação não é "interrogatório de dor"
É exploração natural de consequências. Cliente sente que está pensando junto, não sendo torturado.

### 3. Necessidade-de-Solução vem ANTES da apresentação
Cliente verbaliza o valor antes de você mostrar o produto. Aí apresentação fica fácil — só conecta o que ele já disse com o que você entrega.

### 4. Pergunta aberta > pergunta fechada
"O que está te incomodando?" > "Você está incomodado?"

### 5. Silêncio é amigo
Após pergunta, cale-se. Cliente preenche silêncio com informação valiosa. Vendedor ansioso fala demais e perde insight.

### 6. Nunca pergunte Situação que poderia ter pesquisado
LinkedIn, site, Insta — pesquisa de 5 minutos vale mais que 5 perguntas perdidas.

### 7. SPIN é arte, não ciência rígida
Sequência SPIN não é rígida. Vai e volta. Aprofunda. Recua. Cliente experiente reconhece SPIN bem feito como "uma boa conversa", não como "técnica de venda".

## Como o agente comercial Pinguim usa SPIN

**Em call de venda (humana):**
1. **Pesquisa antes** — LinkedIn, Insta, site do cliente. Reduz S.
2. **Abre com S.1** (validação de baseline)
3. **Faz 1-2 P** pra encontrar dor declarada
4. **Aprofunda com 3-5 I** (mais importante)
5. **Fecha com 2-3 N** pra cliente verbalizar valor
6. **Só DEPOIS apresenta produto** conectando com o que cliente disse

**Em recuperação por mensagem:**
1. Aplica P.1 + I.6 + N.3 (sequência curta)
2. Se há sinal, agenda call

**Sem SPIN, agente faz:**
- Apresenta produto direto → cliente acha caro → deal morre

**Com SPIN, agente faz:**
- Cliente verbaliza dor + valor da solução → apresenta produto que se encaixa → cliente fecha
