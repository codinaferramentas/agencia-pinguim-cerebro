# Challenger Sale — Scripts Canônicos

**Tipo:** metodologia (scripts práticos)
**Pré-requisito:** ler `challenger-sale-principios.md` e `challenger-sale-execucao.md`.

---

## Como usar este documento

Challenger é metodologia de **provocação construtiva** — vendedor traz insight novo que reforma como cliente vê o problema. Scripts foram organizados por momento da venda.

---

## 1. Reframe Statements (Reformulação que reposiciona)

### Script R.1 — Reframe da causa
```
"O que você descreveu como [problema X] geralmente não é o problema real. Na nossa experiência, [reframe — outro problema mais profundo] está por trás."
```

**Quando usar:** abertura de descoberta. Cliente já tem hipótese errada do problema.

**Adaptação Pinguim:**
- "Você descreve o problema como 'falta de tráfego'. Na verdade, o que vejo em 80% dos casos parecidos com o seu é gargalo de conversão — não falta de tráfego."
- "Você acha que precisa de mais conteúdo. Pelos seus números, o problema é distribuição, não produção."

### Script R.2 — Reframe do paradigma
```
"A indústria toda fala que você precisa de [X]. Mas em 2026 o que realmente move resultado é [Y diferente]."
```

**Quando usar:** quando cliente está alinhado com pensamento dominante errado.

**Adaptação Pinguim:**
- "Todo mundo te ensina copy 'persuasiva'. Em 2026, o que converte é copy específica e direta — persuasão floreada queima credibilidade no algoritmo."

### Script R.3 — Reframe baseado em dado
```
"Pesquisamos [N empresas / criadores] como você. Descobrimos que [resultado contra-intuitivo]. Você se identifica com esse padrão?"
```

**Princípio:** dado ou case real é o que dá força ao reframe. Sem dado, vira opinião.

## 2. Provocação Construtiva (não combate)

### Script PC.1 — Discordância respeitosa
```
"Posso discordar de você?"

[espera permissão]

"Pelo que você falou de [X], você está pesando errado. O fator que mais importa pra resultado em [seu nicho] não é [X], é [Y]."
```

**Quando usar:** depois de rapport. Cliente verbalizou pressuposto que está errado.

### Script PC.2 — Pergunta provocativa
```
"E se eu te dissesse que [premissa do cliente] é exatamente o que está te impedindo de crescer?"
```

**Princípio:** força cliente a defender ou reconsiderar pressuposto.

### Script PC.3 — Inversão clássica Challenger
```
"Você tá olhando isso ao contrário. O verdadeiro problema não é falta de [X que cliente acha]. É excesso de [Y que cliente não percebe]."
```

**Adaptação Pinguim:**
- "Você acha que precisa postar mais. Pelo seu engajamento atual, você precisa postar **menos** mas com método específico."

## 3. Insight Delivery (Entrega de insight diferenciado)

### Script ID.1 — Estrutura "Pesquisa → Reframe → Implicação → Solução"

**Estrutura clássica Challenger em 4 partes:**

```
1. [Pesquisa]
"Acabamos de fazer pesquisa com [N] criadores no seu nicho."

2. [Reframe]
"Descobrimos que [resultado contra-intuitivo]. Por exemplo, criadores que [comportamento esperado] têm 40% MENOS engajamento que criadores que [comportamento contraintuitivo]."

3. [Implicação]
"Isso significa que se você continuar [comportamento atual], vai ficar pra trás. Concorrentes que mudarem agora vão te ultrapassar em [tempo]."

4. [Solução]
"O que [agência/método] faz é exatamente o oposto do que [a indústria ensina]. A gente trabalha [novo método específico]."
```

### Script ID.2 — Insight do que NÃO fazer
```
"Os 3 erros mais comuns no seu nicho que ninguém te conta: [erro 1], [erro 2], [erro 3]. Quantos desses você está cometendo?"
```

**Princípio:** apresentar erros antes de soluções. Cliente reconhece, fica receptivo.

## 4. Constructive Tension (Tensão Produtiva)

### Script CT.1 — Não concordar passivamente
```
Cliente: "Acho que minha estratégia tá certa, só preciso de mais ferramenta."
Vendedor: "Antes de você investir em ferramenta, preciso te falar uma coisa que talvez você não queira ouvir. Posso?"
```

### Script CT.2 — Manter opinião sob pressão
```
Cliente: "Discordo. Tenho 5 anos de mercado, sei o que funciona pra mim."
Vendedor: "Respeito sua experiência. E é exatamente por causa dela que talvez você tenha viés de quem vê só uma janela. Posso te mostrar o que dados de 200 outros casos como o seu indicam?"
```

**Princípio Challenger:** vendedor não recua quando cliente discorda. Mantém posição apoiada em dado.

---

## 5. Take Control (Liderar a conversa)

### Script TC.1 — Recusar agenda do cliente
```
Cliente: "Manda só o material por email que eu olho."
Vendedor: "Posso fazer melhor que isso. Em 15 minutos te entrego mais valor que o material consegue. Hoje 14h ou amanhã 10h?"
```

### Script TC.2 — Recusar pedido genérico de desconto
```
Cliente: "Tem desconto?"
Vendedor: "Tenho. Mas antes do desconto, preciso saber se você é o cliente certo pra esse produto. Posso te fazer 3 perguntas?"
```

**Princípio:** Challenger lidera. Não é submisso.

---

## 6. Sequência canônica Challenger (call de descoberta)

```
1. Rapport breve (5min) — Bonding básico
2. Pergunta sobre situação atual
3. [Pesquisa]: "Acabamos de fazer pesquisa com X..."
4. [Reframe]: "Descobrimos que [insight contraintuitivo]..."
5. [Implicação]: "O que isso significa pra você é..."
6. [Tensão]: "Concorda? Discorda?"
7. [Solução]: "É exatamente isso que [nosso método] resolve..."
```

## Princípios universais por trás dos scripts Challenger

### 1. Insight > Empatia
Empatia constrói rapport. Insight constrói autoridade. Challenger prioriza insight.

### 2. Provocação ≠ agressão
Vendedor Challenger discorda **respeitosamente** com dado, não com agressão.

### 3. Não temer perder o deal
Vendedor Challenger prefere perder deal ruim a manter cliente confortável com erro.

### 4. Dado vence opinião
Cada provocação precisa estar apoiada em dado/pesquisa/case. Sem isso, vira opinião que cliente descarta.

### 5. Cliente certo se ressente bem
Cliente bom **adora** ser provocado por alguém com autoridade. Cliente ruim **fecha**. Filtro útil.

## Como o agente comercial Pinguim usa Challenger

**Cenários onde Challenger brilha:**
- Lead que "sabe o que quer" (e está errado)
- Mercado saturado (todos vendem parecido — Challenger diferencia)
- Vendedor com domínio técnico forte do problema
- Cliente cético/conservador que precisa ser provocado

**Cenários onde Challenger falha:**
- Cliente já comprado/positivo (vira agressão desnecessária)
- Vendedor sem autoridade pra provocar (cai mal)
- Cultura corporativa muito hierárquica (provocação ofende)
- Lead emocional (use Voss antes)

**Combinação útil:** Challenger pra abrir conversa (reframe captura atenção), SPIN pra descoberta posterior, MEDDIC pra qualificar.
