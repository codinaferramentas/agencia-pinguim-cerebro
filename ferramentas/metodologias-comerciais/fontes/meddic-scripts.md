# MEDDIC — Scripts Canônicos

**Tipo:** metodologia (scripts práticos)
**Pré-requisito:** ler `meddic-principios.md` e `meddic-execucao.md` antes.

---

## Como usar este documento

MEDDIC não é metodologia conversacional — é framework de qualificação. Os scripts aqui não são "abertura de call" como em SPIN. **São perguntas-chave pra extrair informação dos 6 elementos** durante calls de descoberta, follow-up ou negociação.

Cada script vem com:
- **Que letra MEDDIC ataca** (M, E, D1, D2, I, C)
- **Quando usar** (momento da venda)
- **Como adaptar pra contexto Pinguim** (mentoria/infoproduto)

---

## M — Metrics (Métricas)

Objetivo: extrair número específico que prove ROI da solução.

### Script M.1 — Pergunta direta de baseline
```
Hoje, qual é o seu número atual em [área específica]? Por exemplo, faturamento mensal médio, conversão do funil, ticket médio.
```

**Quando usar:** call de descoberta, depois de entender contexto geral.
**Adaptação Pinguim:**
- Pra closer de programa premium: "qual seu faturamento mensal médio nos últimos 6 meses?"
- Pra recuperação de aluno: "que resultado você teve no último ciclo?"

### Script M.2 — Projeção de resultado
```
Se você resolvesse esse problema dos [X] que você descreveu, qual seria o impacto em número? Quanto a mais por mês? Em quanto tempo?
```

**Quando usar:** depois que cliente verbalizou dor.
**Princípio:** **força o cliente a se comprometer com um número** — esse número vira sua métrica de ROI.
**Adaptação Pinguim:** "Se você dobrasse a conversão do seu desafio, quanto isso significaria por trimestre?"

### Script M.3 — Métrica reversa (custo da inação)
```
E se você não fizesse nada nos próximos 6 meses, o que isso te custa? Em receita perdida, em tempo, em concorrentes ganhando espaço?
```

**Quando usar:** quando cliente está hesitando entre "fazer agora" vs "fazer depois".
**Adaptação Pinguim:** "Cada mês que você passa sem perpétuo é R$ 50K que ficam na mesa. Concorda com essa conta?"

### Script M.4 — Validação da métrica
```
Você acabou de me dizer que [métrica X] vale Y por mês pra você. Se conseguíssemos mover essa métrica em [Z%], esse projeto se paga em quantos meses?
```

**Quando usar:** fim da call de descoberta, antes de propor.
**Por que importa:** transforma a discussão de "preço" em "tempo de payback".

## E — Economic Buyer (Decisor Econômico)

Objetivo: identificar quem libera o cheque (não o influenciador).

### Script E.1 — Pergunta clássica
```
Quando vocês fazem um investimento desse porte, quem normalmente assina embaixo? Você, sócio, conselho?
```

**Quando usar:** começo da call, depois de rapport.
**Adaptação Pinguim:**
- B2B (agência, e-commerce): "quem aprova investimento acima de R$ 10K na sua empresa?"
- B2C (pessoa física): "essa decisão é só sua, ou tem cônjuge/sócio envolvido?"

### Script E.2 — Identificação suave
```
Pra eu te ajudar melhor, me conta: como funciona uma decisão dessas aí? Você decide sozinho, valida com alguém?
```

**Quando usar:** quando cliente é tímido ou pergunta direta soa invasiva.
**Princípio:** disfarça o objetivo (descobrir Economic Buyer) em "quero te ajudar melhor".

### Script E.3 — Pedido pra falar com EB
```
Faz sentido a gente alinhar isso com [nome do EB] também? Posso preparar uma versão direta dos pontos que mais importam pra ele.
```

**Quando usar:** quando você descobre que tem outro decisor mas só está falando com influenciador.
**Adaptação Pinguim:** "Posso preparar um resumo de 5 minutos pra apresentar pro seu sócio? Ele decide ou só você?"

### Script E.4 — Quando o cliente diz "eu decido" mas você desconfia
```
Perfeito, então você é a pessoa final. Pra eu te ajudar a defender essa decisão, o que você precisaria me passar pra eventualmente justificar pra alguém?
```

**Princípio:** mesmo o "decisor único" tem **stakeholders implícitos** (cônjuge, sócio, contador). Esse script extrai a verdade sem confrontar.

## D1 — Decision Criteria (Critérios de Decisão)

Objetivo: descobrir os critérios reais que o cliente vai usar pra decidir.

### Script D1.1 — Pergunta aberta
```
Se você fosse comparar opções pra resolver isso, o que mais pesaria na sua decisão?
```

**Quando usar:** call de descoberta, depois de entender o problema.
**Princípio:** deixa o cliente listar os critérios sem você sugerir.

### Script D1.2 — Pergunta dirigida (top-3)
```
Pelo que eu entendi, você está pesando isso. Pra ti, qual é o ranking dos 3 fatores mais importantes? Preço, metodologia, suporte, tempo, mentor específico, prova social...
```

**Quando usar:** quando cliente respondeu D1.1 vagamente.
**Por que funciona:** força priorização. Cliente nunca tem 8 critérios igualmente importantes — sempre tem top-3.

### Script D1.3 — Validação dos critérios
```
Se eu te entreguei [critério 1] e [critério 2] que você acabou de listar como mais importantes, isso resolveria sua decisão? Ou tem algo mais?
```

**Quando usar:** fim da call, antes de propor.
**Princípio:** descobrir critérios não declarados (objeções escondidas).

### Script D1.4 — Detectar critério oculto
```
Tem alguma coisa específica que faria você falar não, mesmo se a oferta fosse boa? Tipo: política interna da empresa, experiência ruim no passado, alguém que você confia falar pra esperar?
```

**Princípio:** descobre **gatilhos de "não"** que o cliente não verbaliza espontaneamente.

---

## D2 — Decision Process (Processo de Decisão)

Objetivo: mapear o roteiro concreto de como o cliente vai decidir.

### Script D2.1 — Mapeamento simples
```
Me ajuda a entender o processo: depois dessa nossa conversa, quais são os próximos passos do seu lado? Quem você precisa envolver, em quanto tempo?
```

**Quando usar:** fim de call de descoberta.
**Adaptação Pinguim:** "Pra você fechar comigo, o que precisa acontecer? Falar com cônjuge, alinhar agenda, ver se tem orçamento?"

### Script D2.2 — Cronograma reverso
```
Quando você gostaria de estar [resultado X — vendendo perpétuo, com Reels rodando, com squad trabalhando]? Pra chegar lá em [data], a gente precisaria começar até quando?
```

**Quando usar:** pra criar urgência **interna** (não vendedor pressionando — cliente percebendo que está atrasado).

### Script D2.3 — Identificar bloqueios
```
O que pode atrapalhar essa decisão andar? Tem algo que costuma travar — agenda lotada, fim de mês, decisão dependente de outra?
```

**Quando usar:** depois de mapear processo.
**Princípio:** **antecipa as objeções de execução**. Vendedor que não pergunta isso é surpreendido depois.

### Script D2.4 — Compromisso de próximo passo
```
Combinado então: você vai falar com [pessoa] até [dia], e a gente se fala [dia + 2] às [hora]? Posso te mandar lembrete?
```

**Por que importa:** **fecha o próximo passo concreto na própria call**. Sem isso, deal fica em limbo.

## I — Identify Pain (Identificar Dor)

Objetivo: ancorar a venda na dor primária do cliente, não em features do produto.

### Script I.1 — Dor declarada
```
O que te fez procurar uma solução agora? O que está doendo?
```

**Quando usar:** abertura da call.
**Adaptação Pinguim:** "Você poderia ter feito isso há 6 meses, há 1 ano. Por que agora?"

### Script I.2 — Aprofundar dor
```
Quanto isso te incomoda numa escala de 1 a 10? Por que não 5? Por que não 9?
```

**Princípio Sandler aplicado a MEDDIC:** força o cliente a verbalizar a intensidade. Dor 6 raramente fecha; dor 8+ fecha.

### Script I.3 — Custo da dor
```
Se essa situação continuar mais 6 meses, o que acontece? Você perde o quê em concreto?
```

**Quando usar:** quando cliente reconhece dor mas não verbaliza urgência.
**Por que funciona:** transforma "estou incomodado" em "estou perdendo R$ X".

### Script I.4 — Causa raiz
```
Pelo que você descreveu, parece que o problema raiz é [hipótese]. Faz sentido? Ou tem outra coisa por trás?
```

**Princípio:** confirma diagnóstico antes de propor solução. Vendedor amador propõe direto; vendedor MEDDIC valida primeiro.

## C — Champion (Defensor Interno)

Objetivo: identificar e qualificar a pessoa que vai vender por você internamente.

### Script C.1 — Identificar Champion
```
Quem da sua equipe / da sua casa / do seu círculo está mais animado com essa possibilidade? Quem sentiria mais a transformação?
```

**Adaptação Pinguim:**
- B2B: "qual sócio / funcionário se beneficia mais disso?"
- B2C: "tem alguém perto de você que ia comemorar se você fizesse acontecer?"

### Script C.2 — Qualificar Champion
```
Esse [nome do Champion] tem influência real na decisão? Ele tem voz, é ouvido, ou é mais um apoio moral?
```

**Princípio MEDDIC:** Champion ≠ Coach. Coach passa info. Champion **vende** por você. Sem essa qualificação, vendedor confunde simpatia com poder.

### Script C.3 — Capacitar Champion
```
Se eu te entregasse os pontos-chave em formato pronto pra você apresentar [pra sócio, pra cônjuge, pro EB], isso ajudaria? O que você precisaria pra fazer essa conversa internamente?
```

**Quando usar:** quando você confirmou que tem Champion mas não está com EB direto.
**Por que importa:** transforma o lead em **vendedor interno**. Você arma ele com argumentos prontos.

### Script C.4 — Detectar falso Champion
```
Se essa decisão dependesse 100% de você, você diria sim hoje?
```

**Princípio:** se a pessoa não diria sim mesmo com poder pleno, ela não é Champion — é só simpatizante.

## Scripts compostos (cobrem múltiplas letras)

### Script Composto 1 — Sequência de fechamento de call
```
1. "Pra fechar nossa conversa: o que mudou na sua cabeça nos últimos 30 minutos?" (I)
2. "Pelo que entendi, o critério mais importante pra ti é [X], certo?" (D1)
3. "E pra você dar próximo passo, depende de quem mais?" (E + D2)
4. "Faz sentido a gente se falar [dia + 3] depois que você [próximo passo]?" (D2 commitment)
```

### Script Composto 2 — Recuperação de deal frio
```
1. "Faz X tempo que a gente conversou. O que mudou desde lá?" (I)
2. "O que está te impedindo de avançar?" (D1 + objeção real)
3. "Se a gente removesse esse bloqueio, você avançaria? Ou tem outra coisa?" (D1 oculto)
4. "Quem mais precisa estar nessa decisão pra rolar agora?" (E)
```

### Script Composto 3 — Negociação de objeção de preço
```
1. "Antes de falar de preço — você está convicto que isso resolve seu [dor X]?" (I)
2. "Pelo que você me disse, [resultado] vale [Y por mês] pra ti, certo?" (M)
3. "Então a discussão não é se vale, é como cabe agora. Qual é a real?" (E + D2)
```

---

## Princípios universais por trás dos scripts

1. **Pergunta aberta antes de fechada.** "O que pesa mais?" antes de "é o preço?".
2. **Validar antes de propor.** Toda hipótese sobre dor/critério/processo precisa ser confirmada pela boca do cliente.
3. **Próximo passo sempre concreto.** Toda call termina com data + ação combinada.
4. **MEDDIC é diagnóstico, não interrogatório.** Aplicar as 6 letras de forma natural na conversa, não como checklist mecânico.
5. **Falta resposta = deal não qualificado.** Se faltou M, ou E, ou D1, ou D2, o deal **não merece** virar proposta. Volta pra descoberta.

---

## Como o agente comercial Pinguim usa MEDDIC scripts

**Cenário típico:** Closer humano em call de fechamento de programa premium (Lyra, Taurus, Orion).

1. Closer abre call com I.1 (dor declarada)
2. Aprofunda com I.2 e I.3
3. Extrai métrica com M.1 e M.2
4. Mapeia EB com E.1 ou E.2
5. Descobre critérios com D1.1 e D1.2
6. Mapeia processo com D2.1 e D2.2
7. Identifica Champion com C.1
8. Fecha próximo passo com Script Composto 1

Cada call gera um **MEDDIC Score** documentado no CRM. Deal com 6/6 vai pra proposta. Deal com <6/6 volta pra descoberta antes de propor.
